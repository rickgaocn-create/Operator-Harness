# Skill: Vault Sanity Guardian (`/sanity`)

Audit the {{USER_NAME}} vault for drift from architecture, broken cross-pillar links, frontmatter rot, stale workstreams, and task-hygiene risks (overdue / orphaned cards in Kanbans). Tiered-authority audit: auto-fixes the safe, proposes the risky. Invoked manually or scheduled (e.g., Friday 8am as pre-weekly-review).

The job: keep the vault lean and high-leverage. Surface drift; auto-fix the safe; propose the risky. Never overstep — this is the user's primary knowledge base.

## When to Use

- User says `/sanity`, `/sanity --propose-only`, or `/sanity --scope <folder1,folder2>`
- Before `/weekly-update` to preempt drift findings
- After heavy ingestion sessions (multiple `/source-ingest` runs)
- After bulk task captures in `06 Tasks/Inbox.md` (verify no runaway sync)
- After structural changes (folder rename, rule-layer edit, new skill added)

---

## Voice

Strategic + slightly witty, matching CLAUDE.md tone. Findings read as decisions to consider, not violations to fix. No "VIOLATION:" or AI-fluff. Density over prose.

Examples:
- ✅ *"Card-graph orphan rate up 12% from last pass — likely from the EPIC ingest burst. Worth chaining /card-lint?"*
- ✅ *"Three Action files in `12 Active/` haven't moved in 3+ weeks. `13 Maybe/` candidates or quietly archive?"*
- ❌ *"VIOLATION: 02 Cards/{{PROJECT_A}}/C260424-foo.md missing required frontmatter `source-note`"*
- ❌ *"It is important to maintain frontmatter integrity..."*

---

## Operating Principles

1. **Read before write.** A full scan happens before any patch. Findings are clustered before any auto-fix runs.
2. **Auto-fix is conservative; propose is everything else.** When in doubt, propose.
3. **The rule layer wins.** When `09 Rules/*.md` and a skill disagree, the rule file is canonical.
4. **Sensitive content is read-only.** Deal artifacts, contracts, prose body of meeting notes/cards/projects — never edit. Frontmatter and meta-files only.
5. **The user has final say.** A `lint-exempt: true` field on any file makes the skill skip it.

---

## Severity Tiers

| Symbol | Meaning | Default action |
|---|---|---|
| 🔴 Critical | Breaks the graph or a contract (broken wikilinks, missing required frontmatter, duplicate-title across same Kanban lane) | Auto-fix if in safe list, else propose with high priority |
| 🟡 Warning | Degrades quality but graph still functional (stale lifecycle, orphan, asymmetric link, frontmatter incomplete) | Propose |
| 🟢 Suggestion | Optimization / observation (hub formation, headline drift, concept frequency) | Log only — no action requested |

A run with 0🔴 / 0🟡 / N🟢 is healthy. Don't chase 0/0/0 — some 🟢 is accepted entropy.

## Auto-Fix Authority Matrix

The skill may auto-edit ONLY for findings in the "Auto" column. Everything else is propose-then-execute (surface to the user, wait for explicit approval).

| Finding | Auto / Propose |
|---|---|
| Daily note missing `> Parent: [[YYYY-Www]]` blockquote at top (computable from filename) | **Auto** |
| Daily/weekly/12-week file missing required frontmatter fields where computable from filename (`type`, `date`, `weekday`, `week`) | **Auto** |
| Action file missing `## Linked Tasks` Dataview scaffold (when `chain-anchor:` exists) | **Auto** |
| Dead wikilink in `01 Wiki/` (target file confirmed missing) | **Auto** — comment out as `<!-- broken: [[old-name]] -->`, preserve original text |
| Trailing whitespace / mixed line endings in agent-managed files | **Auto** |
| `(C) ` prefix missing on file with `created-by: claude` frontmatter | **Auto** rename |
| Anything else | **Propose** |

When in doubt → propose, not auto.

## Phase 2: Audit by Folder

Use Read for content, Glob for file enumeration, Grep for pattern search, Bash for `ls -lat` / `find` / `wc -l` / `date`.

### `00 Raw/`

```bash
find "00 Raw/" -maxdepth 1 -type f -name "*.md"
find "00 Raw/Clippings/" -maxdepth 1 -type f -name "*.md"
```

For each file:
- **Stuck (>14d in `00 Raw/` root):** 🟡 — propose `/inbox-process` candidate, suggest destination if frontmatter hints
- **Stuck (>7d in `00 Raw/Clippings/`):** 🟡 — propose `/source-ingest` candidate
- **Empty/near-empty (0 bytes or frontmatter only):** 🟡 — flag for cleanup
- **Legacy path drift** (`00 Raw/Daily/` exists with files): 🟡 — propose move to `04 Notes/daily notes/`

### `01 Wiki/`

```bash
find "01 Wiki/" -name "*.md" -not -path "*/scholar/*"
```

For each file:
- **Dead wikilinks:** 🔴 → **AUTO** comment out as `<!-- broken: [[name]] -->`
- **Missing `(C)` prefix when `created-by: claude` in frontmatter:** 🟡 → **AUTO** rename
- **Stale decision block** — search for `**当前决策（YYYY-MM-DD）**` patterns; if date >90d old AND counterparty has recent meeting notes referencing it: 🟡 propose decision-block refresh
- **Orphan archives** — file with zero inbound wikilinks: 🟡

### `02 Cards/`

Run inline equivalent of `/card-lint` Phase 1–4 (Frontmatter integrity, Body anatomy, Graph health, Lifecycle staleness, Domain coherence, Source ledger). Severity tags match `/card-lint`'s output.

Plus cross-pillar check:
- **Cross-pillar reference rot** — Card cites Action file in `10 Action/_archive/`. If archive happened in last 30d → 🟡. Older → 🟢 informational.
- **Card-Action harvest gap** — Action file `status: done` in `_archive/` with no `## Linked Cards` section: 🟡

**Authority: NONE.** All Card findings → surface to user; user invokes `/card-lint` Phase 5+ manually to act.

### `03 Projects/`

For each project subfolder (skip `(PROJECT TEMPLATE)`):
- **Read project's `CLAUDE.md`** — compare against current `/new-project` output:
  - 🟡 References `00 Raw/Daily/` or other legacy paths
  - 🟡 References "KJ OS" or other pre-3-pillar terminology
  - 🟡 Restates Task Capture Protocol inline (now in `[[09 Rules/tasks.md]]`)
  - 🟡 No mention of 3-pillar architecture (Card / Time / Action)
  - 🟡 `Last updated:` >30d AND project status = "Active"
- **Pipeline subfolder structure** — compare against canonical from root CLAUDE.md:
  - {{PROJECT_A}}: `00 Prospects → 01 Pitches → 02 Negotiations → 03 Contracts → 04 Delivery`
  - {{ORG_B}}: `00 Sourcing → 01 Screening → 02 Diligence → 03 Proposals → 04 Board → 05 Closed Deals`
  - 🟡 Missing or oddly-named folders
- **Project `COMMANDS.md` drift** — entries listing skills not in `<vault>/.claude/skills/` or `~/.claude/skills/`: 🟡
- **Cross-pillar link rot** — `[[02 Cards/...]]` / `[[10 Action/...]]` / `[[04 Notes/...]]` pointers from project files where target missing: 🟡

**Authority: NONE.** All findings propose-only.

### `04 Notes/`

```bash
find "04 Notes/daily notes/" -name "*.md"
find "04 Notes/weekly/" -name "*.md"
find "04 Notes/12-week/" -name "*.md"
```

For each daily:
- **Missing `> Parent: [[YYYY-Www]]`** at top: 🔴 → **AUTO** insert
- **Missing required frontmatter** (`type`, `date`, `weekday`, `parent`): 🔴 → **AUTO** insert what's computable from filename; flag the rest
- **Frontmatter `parent:` doesn't match filename's ISO week:** 🔴 → propose fix

For each weekly:
- **Missing required frontmatter** (`type`, `week`, `start`, `end`, `parent`, `status`): 🔴 → AUTO what's computable

For each 12-week:

- `04 Notes/Daily/` (capital D) exists with files: 🔴 propose migration to canonical `04 Notes/daily notes/`

**Skip:** `04 Notes/Meetings/` and `04 Notes/Session Logs/` prose — read for cross-references but never edit body.

### `.claude/skills/<name>/SKILL.md`

For each skill file:
- **Missing YAML frontmatter** (no `name:` or `description:`): 🔴 propose frontmatter scaffold
- **Dead `/skill-X` reference** — body cites `/skill-X` but `.claude/skills/skill-X/SKILL.md` doesn't exist (check both vault and `~/.claude/skills/`): 🔴 propose hand-off rewrite
- **Stale `05 Skills/` path reference** — body still cites legacy `05 Skills/...` path post-2026-05-11 migration: 🟡 propose path update
- **Restates rule-layer content** — body contains a full file-routing or task-format spec that now lives in `09 Rules/`: 🟡 propose consolidation
- **Edit timestamp >90d** AND skill mentions deprecated patterns: 🟢 informational

**Authority: NONE.** Skills are user-authored; surface, user decides.

### `06 Tasks/` + `03 Projects/*/Tasks.md` (task surfaces)

## Phase 4: Apply Auto-Fix Patches

For findings in the **Auto** column, apply patches in a single batch. Use `Edit` (not `Write`) for surgical patches.

**Hard rules during auto-fix:**
- Each patch is one `Edit` call with `old_string` + `new_string` — no bulk rewrites
- Never touch `06 Tasks/` (hard-coded)
- Never modify a Card's H1 / blockquote / Mechanism / Application sections
- Never edit prose body of: meeting notes, session logs, contracts, deal memos, board decks
- Never modify `created:` frontmatter field on any file (permanent provenance)
- If an auto-fix target file has been modified within last 5 minutes (likely active user editing): skip and add to propose list

After auto-fix batch: re-scan affected files to confirm patches stuck. Any file where the patch failed → escalate to propose-only finding.

## Hard Rules

- **`06 Tasks/` and `03 Projects/*/Tasks.md` task content is read-only by sanity.** Hard-coded. Audit only — never edit task lines.
- **Prose body of sensitive files is read-only.** Meeting notes, session logs, contracts, deal memos, board decks, Card Mechanism/Application sections — never edit.
- **`lint-exempt: true` is honored everywhere.** A file with this frontmatter field is skipped entirely.
- **`created:` frontmatter is immutable.** Permanent provenance stamp.
- **Auto-fixes are surgical.** One `Edit` call per patch, never bulk rewrites.
- **Don't touch `_archive/` or `_processed/`.** Read for context (cross-pillar reference checks); never edit.
- **Don't auto-spawn files.** Only Edit existing files for auto-fixes. New file creation (sanity report, rolling log header) is the only Write.
- **CLAUDE.md / MEMORY.md / GOALS.md edits require per-edit authorization.** Never auto-fix these three files. Each proposed edit gets shown to the user as a single yes/no — no batching.
- **One run per day max** — if today's sanity report exists, ask user before overwriting.

## Failure Modes

| Symptom | Fix |
|---|---|
| First run produces 100+ findings | Healthy — initial vault debt. Triage by severity, fix 🔴 only on this pass, schedule a second pass next week for 🟡. |
| Same warning persists across 3 runs | User has consciously declined to fix. Stop surfacing it; suggest adding `lint-exempt: true` or a "won't fix" entry. |
| Auto-fix batch fails on a file | Skip and escalate to propose-only. Don't retry — likely a permission or concurrent-edit issue. |
| Folder structure has changed since rules layer was written | Surface to user with proposed rules-layer update (don't auto-edit `09 Rules/`). |
| Dead-link auto-fix would comment out a wikilink that's just temporarily missing | Skip and re-flag next pass. Don't comment out something that exists. |
| User invokes `/sanity` while a session is mid-task | Run the audit but defer write phase — show findings, ask "apply auto-fixes now?" Yes/no. |

## Cadence Awareness

The skill runs in two modes:

**Manual (`/sanity`):**
- User-invoked, ad-hoc
- Apply auto-fixes immediately if user is present and engaged
- Generate full report

**Scheduled (Friday 8am — pre-weekly-review):**
- Apply auto-fixes immediately (low-risk by design)
- Surface 🔴 findings via daily note + rolling log
- User reads report Friday morning before EOD weekly review

If today is Friday and the user invokes `/sanity` manually: skip the scheduled run (avoid double-pass).
