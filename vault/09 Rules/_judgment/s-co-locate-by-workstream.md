# Structure Principle · Co-locate by workstream

**Principle:** Organize by workstream / relationship (the `chain-anchor`), not by abstract process stage. Files that move together in real work should live together.

**Why it's taste:** this is already the vault's *content* philosophy — `chain-anchor` binds Action ↔ Task ↔ Card across one workstream ([[09 Rules/action.md]]). The funnel-by-stage scheme (`Prospect`/`Negotiate`/`Deliver`) fragments a single deal across stage-folders and fights how the work actually flows; the structure layer should inherit the same co-location logic the content layer already won on.

**Tells when honored:** a deal's artifacts are findable in one place; structure mirrors live workstreams.
**Tells when violated:** one partnership's files scattered across five stage-folders; structure mirrors a pipeline nobody walks.

**Operationalized by:** `chain-anchor` ([[09 Rules/action.md]]); future vault-manager reorg proposals.
