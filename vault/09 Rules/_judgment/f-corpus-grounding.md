# Framework · Corpus grounding (vault-first)

**Procedure:** Before answering, guessing, or acting, retrieve from your own corpus first — Glob/Grep the vault (Wiki / Cards / Rules / OKR / project files) for what is already written. Never let the model fall back to training-data inference when the answer is already in your record. Failure order: vault grep → public knowledge → ask the user → only then assume.

**When to apply:** Any unfamiliar entity; any plan or decision review; any daily / Top-3 framing; any "what's the state of X" — before the first substantive response.

**Why it embodies these values:** grounding in your accumulated record over a comfortable guess is truth-over-comfort applied to *inputs* (`v-truth-over-comfort`); and claiming/adopting only what's actually in the record — not generic LLM priors — keeps the frontier grounded in real data (`v-frontier-grounded`).
