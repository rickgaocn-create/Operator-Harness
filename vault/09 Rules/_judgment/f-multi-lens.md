# Framework · Multi-lens evaluation (Head-of-X)

**Procedure:** Pressure-test an artifact through 1–3 declared functional lenses (Ops / Finance / Strategy / BD / Risk / Product / People) chosen by what the artifact contains. Declare the lenses up top; never run them silently. Each lens watches for its own failure class (unit economics, single-point-of-failure, defensibility, term asymmetry, etc.).

**When to apply:** Evaluating any business artifact — deals, vendor demos, OKRs, proposals.

**Why it embodies these values:** multiple independent lenses surface the blind spot a single view flatters past (`v-truth-over-comfort`), and each lens lands on a concrete decision driver (`v-pragmatic-ambition`).
