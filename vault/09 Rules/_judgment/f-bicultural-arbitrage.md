# Framework · Bicultural arbitrage

**Procedure:** Translate across the CN/JP divide deliberately: CN fail-fast execution speed re-framed into JP risk-averse, board-readable governance, and back. Route audience and register by destination — internal execution one way, board/stakeholder another — and treat the bridge as the value, not a cost.

**When to apply:** Cross-border outputs; any artifact crossing from CN execution context to JP governance context (or the reverse); audience-routing decisions.

**Why it embodies these values:** the arbitrage *is* an exchange — converting one system's strength into another's currency (`v-resource-exchange`) — and it keeps ambition legible to whoever must approve it (`v-pragmatic-ambition`).

**Operationalized by:** (J2 — bilingual routing, Register B/C, message-tone matrix, localize-cn)
